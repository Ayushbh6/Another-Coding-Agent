# ACA: Another Coding Agent ;)

**The local-first, production-grade coding assistant.**

ACA is a sophisticated agentic system designed to drive development alongside you. Built with a focus on precision, transparency, and safety, ACA goes beyond simple chat by employing a multi-agent architectural loop (Master Architect, Challenger, and Worker) to ensure high-quality code implementation.

---

## 🚀 Features

- **Multi-Agent Orchestration**: Every implementation request passes through a "Master Architect" for planning, a "Challenger" for critique and security auditing, and a specialized "Worker" for execution.
- **Local-First Execution**: Deep integration with your workspace for file reading, code searching, and terminal execution.
- **Surgical Code Edits**: Uses atomic, context-aware patching to modify large files precisely without overwriting unrelated code.
- **Human-in-the-Loop Safety**: Every system-level mutation (file writing, shell commands) requires your explicit permission.
- **Universal LLM Support**: Powered by OpenRouter for access to any cutting-edge model (Gemini, Claude, GPT-4, etc.).

## 🏗️ Multi-Agent Loop

ACA operates on a feedback loop to guarantee quality:

1.  **Classification**: The system identifies if your request is casual chat, deep analysis, or an implementation task.
2.  **Architectural Planning**: The **Master Architect** breaks down complex requests into actionable, sequential steps.
3.  **Adversarial Review**: The **Challenger** (Lead Staff Engineer persona) audits the plan for edge cases, security risks, and regressions.
4.  **Autonomous Execution**: The **Worker** executes the finalized plan using specialized workspace tools.
5.  **Synthesis**: A final report is generated by a Technical Lead summarising the outcome and any remaining risks.

## 🛠️ Tooling

ACA comes equipped with "Production-Grade" workspace tools:
- `search_code`: Fast, pattern-based repository searching.
- `read_file` / `list_files`: Contextual awareness of your project structure.
- `apply_patch`: Surgical, exact-match code modification.
- `run_command`: Directly interface with your build and test suits.

## 🚦 Quick Start

### 1. Configure Environment
Create a `.env` file with your OpenRouter API key:
```env
OPENROUTER_API_KEY=your_key_here
```

### 2. Launch ACA
```bash
source .venv/bin/activate
python main.py
```
